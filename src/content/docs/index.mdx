## Why Learn Lua with TDD?

Lua is a beautifully minimal and lightweight language, making it highly extensible but also easy to misuse if left unverified. This project bridges that gap. By writing tests before you write implementation code, you build a deep, mechanical understanding of Lua’s syntax while internalizing robust software engineering practices.

Whether you are targeting embedded environments, automation scripting, or high-performance game engines, a strict test-driven workflow ensures your codebase stays maintainable as complexity scales.

## The Philosophy Behind This Guide

Traditional learning resources usually fail software teams in one of two ways: they either demand heavy, dense textbook reading that quickly loses momentum, or they rely on isolated logic puzzles (katas) that don't scale to real-world application design.

When you're trying to move a team or yourself from syntax curiosity to production-ready software engineering, ad-hoc experimentation becomes unstructured and confusing. 

### A Functional Approach
The most effective way to master a technical language is through structured, interactive mechanics. This guide approaches programming like an engineering discipline:
*   **Isolate concepts** into small, digestible milestones.
*   **Verify assumptions** immediately inside an isolated test environment.
*   **Document features** implicitly through the code assertions you write.

Instead of guessing how a metatable behaves or messing around indefinitely in a REPL, you will write a test that proves exactly how the system handles data.

## Who This Is For

*   **Lua Newcomers:** Developers looking to learn the language mechanics through a structured, highly disciplined approach.
*   **Experienced Devs:** Programmers who know basic Lua syntax but want to transition to a professional, test-first design workflow.

## Prerequisites

*   A local development machine with [Lua](https://www.lua.org/) (5.x or LuaJIT) installed.
*   A terminal environment and your preferred text editor.
*   Basic programming foundations (loops, conditional blocks, functions, and variables).
